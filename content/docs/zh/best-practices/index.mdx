# 最佳实践

欢迎来到 最佳实践 专区！在这里您将找到关于 最佳实践 的宝贵资源和指南。

## 社区共识的最佳实践（按被提及频率排序）

- **维护 `CLAUDE.md` 与记忆**：在仓库根目录（以及子目录/用户目录）维护精炼的 `CLAUDE.md`；用 `#` 快速沉淀命令、规范与重要规则。
- **先规划再编码**：先让 Claude 调研并给出方案（探索 → 规划 → 编码 → 提交）；用增强思考（“think hard/ultrathink”）；避免一次性超大指令。
- **保持上下文专注**：任务间多用 `/clear` 开新会话；在自然断点用 `/compact`；不同任务尽量分开线程。
- **测试优先且由人书写**：倾向 TDD；每次小改动后立刻运行测试/类型检查/格式校验；避免让 AI 修改关键测试或断言。
- **权限与安全**：配置允许列表；谨慎使用自动批准；仅在受限环境且有备份时考虑 YOLO（`--dangerously-skip-permissions`）。
- **明确且具体的指令**：给出精确需求、边界条件与成功标准；显式引用文件；必要时附图片/URL。
- **小步快跑、可回滚**：小范围迭代与审阅；及时中断纠偏与撤销；避免一次性大变更。
- **用子代理与工作树并行**：利用 Task/子代理、多实例、多个检出或 git worktree 并行独立任务。
- **定制环境**：自定义斜杠命令与 Hooks；优化 `.claude/settings.json`；安装 `gh` CLI；启用 IDE/终端集成。
- **善用 Claude 操作 git/GitHub**：生成提交/PR，修复 CI 失败，分流与归类 Issue，辅助复杂 git 操作。
- **无头/CI 自动化**：在脚本与 CI 中使用无头模式执行检查、分流或结构化流程。
- **划定“禁止触碰”区域**：迁移脚本、API 契约、机密配置与安全关键代码由人掌控，并加边界提示。
